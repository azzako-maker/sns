# 📱 반응형 테스트 가이드

이 문서는 SNS 프로젝트의 반응형 디자인을 테스트하는 방법을 안내합니다.

---

## 🖥️ Desktop (1024px 이상)

### 테스트 환경

- **브라우저**: Chrome, Firefox, Safari, Edge
- **화면 크기**: 1024px 이상 (일반적으로 1920x1080, 1440x900 등)
- **디바이스**: 데스크톱, 노트북

### 확인 사항

#### 1. Sidebar (244px 너비)

- [ ] 좌측에 244px 너비의 Sidebar가 표시됨
- [ ] 아이콘과 텍스트가 함께 표시됨
- [ ] 메뉴 항목: 홈, 검색, 만들기, 프로필
- [ ] Hover 시 배경색 변경 (#FAFAFA)
- [ ] Active 상태 표시 (현재 경로에 맞는 메뉴 하이라이트)
- [ ] Clerk UserButton이 하단에 표시됨

#### 2. PostCard (최대 너비 630px)

- [ ] 게시물 카드가 최대 630px 너비로 중앙 정렬됨
- [ ] 배경색 #FAFAFA 적용됨
- [ ] 게시물 이미지가 1:1 정사각형 비율 유지
- [ ] 헤더, 액션 버튼, 컨텐츠 영역이 올바르게 표시됨

#### 3. 게시물 상세 모달

- [ ] 게시물 이미지 클릭 시 모달이 열림
- [ ] 모달 크기: 최대 너비 900px
- [ ] 좌측 50%: 이미지 영역
- [ ] 우측 50%: 댓글 영역
- [ ] 반응형으로 동작 (화면이 작을 때 세로 스택)

#### 4. 프로필 페이지

- [ ] 프로필 이미지: 150px 원형
- [ ] 프로필 정보가 가로로 배치됨
- [ ] PostGrid가 3열 그리드로 표시됨

## 📱 Tablet (768px ~ 1023px)

### 테스트 환경

- **브라우저**: Chrome, Firefox, Safari
- **화면 크기**: 768px ~ 1023px (일반적으로 iPad, 태블릿)
- **디바이스**: 태블릿, 작은 노트북

### 확인 사항

#### 1. Icon-only Sidebar (72px 너비)

- [ ] 좌측에 72px 너비의 Sidebar가 표시됨
- [ ] 아이콘만 표시되고 텍스트는 숨김
- [ ] Hover 시 툴팁 표시 (선택사항)
- [ ] Active 상태 표시

#### 2. PostCard 반응형

- [ ] 게시물 카드가 화면 너비에 맞게 조정됨
- [ ] 패딩이 적절하게 조정됨
- [ ] 모든 콘텐츠가 올바르게 표시됨

#### 3. 게시물 상세 모달

- [ ] 모달이 화면 크기에 맞게 조정됨
- [ ] 좌우 분할 또는 세로 스택으로 전환됨

---

## 📱 Mobile (< 768px)

### 테스트 환경

- **브라우저**: Chrome, Safari, Firefox
- **화면 크기**: 320px ~ 767px (일반적으로 스마트폰)
- **디바이스**: iPhone, Android 스마트폰

### 확인 사항

#### 1. Header (상단, 60px 높이)

- [ ] 상단에 60px 높이의 Header가 고정됨
- [ ] 로고가 좌측에 표시됨
- [ ] 우측에 알림/DM/프로필 아이콘 표시
- [ ] 스크롤 시에도 상단에 고정됨

#### 2. BottomNav (하단, 50px 높이)

- [ ] 하단에 50px 높이의 BottomNav가 고정됨
- [ ] 5개 아이콘: 홈, 검색, 만들기, 좋아요, 프로필
- [ ] Active 상태 표시
- [ ] 스크롤 시에도 하단에 고정됨

#### 3. Sidebar 숨김

- [ ] Sidebar가 완전히 숨겨짐
- [ ] Header와 BottomNav만 표시됨

#### 4. 전체 너비 PostCard

- [ ] 게시물 카드가 화면 전체 너비 사용
- [ ] 좌우 패딩만 적용 (일반적으로 0 또는 최소)
- [ ] 이미지가 화면 너비에 맞게 조정됨

#### 5. 게시물 상세 페이지 (전체 페이지)

- [ ] 게시물 이미지 클릭 시 전체 페이지로 이동
- [ ] 모달이 아닌 전체 페이지 형태
- [ ] 뒤로 가기 버튼 동작 확인

## 🧪 테스트 방법

### 1. 브라우저 개발자 도구 사용

#### Chrome DevTools

1. **F12** 또는 **우클릭 → 검사**로 개발자 도구 열기
2. **Ctrl+Shift+M** (또는 디바이스 툴바 아이콘)으로 반응형 모드 활성화
3. 화면 크기 선택:
   - Desktop: 1920x1080, 1440x900 등
   - Tablet: iPad (768x1024), iPad Pro (1024x1366)
   - Mobile: iPhone SE (375x667), iPhone 12 Pro (390x844), Galaxy S20 (360x800)

#### Firefox DevTools

1. **F12**로 개발자 도구 열기
2. **Ctrl+Shift+M**으로 반응형 디자인 모드 활성화
3. 화면 크기 선택 또는 직접 입력

### 2. 실제 디바이스 테스트

가능하면 실제 디바이스에서 테스트하는 것이 가장 정확합니다:

- **Desktop**: 실제 데스크톱 또는 노트북
- **Tablet**: iPad, Android 태블릿
- **Mobile**: iPhone, Android 스마트폰

### 3. 온라인 테스트 도구

- [Responsive Design Checker](https://responsivedesignchecker.com/)
- [BrowserStack](https://www.browserstack.com/)
- [LambdaTest](https://www.lambdatest.com/)

## ✅ 테스트 체크리스트

### 기능 테스트

- [ ] 모든 링크가 정상 작동
- [ ] 모든 버튼이 정상 작동
- [ ] 폼 제출이 정상 작동
- [ ] 모달 열기/닫기가 정상 작동
- [ ] 무한 스크롤이 정상 작동

### 레이아웃 테스트

- [ ] 텍스트가 잘리지 않음
- [ ] 이미지가 비율 유지하며 표시됨
- [ ] 모든 요소가 화면에 잘 보임
- [ ] 스크롤이 정상 작동
- [ ] 고정 요소(Header, BottomNav)가 올바른 위치에 고정됨

### 성능 테스트

- [ ] 페이지 로딩 속도가 빠름
- [ ] 이미지 로딩이 최적화됨
- [ ] 스크롤이 부드럽게 동작
- [ ] 애니메이션이 부드럽게 동작

### 접근성 테스트

- [ ] 키보드 네비게이션이 가능
- [ ] 스크린 리더와 호환
- [ ] 충분한 색상 대비
- [ ] 터치 영역이 충분히 큼 (모바일)
